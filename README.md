# bye-tcp-internet

**Industrial-grade TCP/IP baseline profile for modern Windows 10/11 networking stacks.**

Проект представляет собой верифицированный профиль настроек реестра для сетевого стека `tcpip.sys`. Цель — оптимизация базовых параметров Windows для обеспечения стабильной работы в современных сетях (2026) без использования устаревших или деструктивных методов.

---

## Философия проекта

Данный профиль **не является** «магическим ускорителем интернета». Это набор консервативных и актуальных переопределений (registry overrides), которые по-прежнему учитываются современным стеком Windows (10 22H2 / 11 / LTSC 2021+).

### Принципы разработки:

* **No Legacy:** Исключены параметры времен Windows XP/7.
* **No Placebo:** Только ключи, имеющие документально подтвержденное влияние на `tcpip.sys`.
* **Minimalism:** Изменяются только те параметры, где стандартные значения Windows могут приводить к деградации производительности в высоконагруженных или нестабильных средах.
* **Transparency:** Полное соответствие современным стандартам адаптивного сетевого стека.

## Техническая спецификация

### Что внедряет данный профиль:

1. **PMTU Discovery:** Гарантированная активация механизма определения максимального размера пакета для минимизации фрагментации.
2. **Black Hole Detection Removal:** Отключение устаревших механизмов поиска «черных дыр», которые могут вызывать ложные задержки.
3. **SACK + Window Scaling:** Принудительное удержание активными механизмов выборочного подтверждения и масштабирования окна.
4. **Ephemeral Port Range:** Расширение диапазона динамических портов для предотвращения дефицита сокетов при большом количестве параллельных соединений.
5. **TIME_WAIT Optimization:** Сокращение времени удержания закрытых соединений (TcpTimedWaitDelay) для их быстрой переработки.
6. **Multimedia Throttling Removal:** Отключение сетевого троттлинга со стороны планировщика задач (NetworkThrottlingIndex).
7. **DNS Negative Caching:** Стабилизация поведения кэша при неудачных запросах.

### Чего этот профиль НЕ делает (Scope Out):

* **Autotuning:** Не отключает автоподстройку окна (Windows лучше справляется с этим динамически).
* **Congestion Control:** Не меняет CUBIC на устаревшие алгоритмы.
* **TCPNoDelay:** Не навязывает глобальный Nagle Algorithm override (это задача прикладного ПО).
* **Deprecated Keys:** Не содержит ключей `TcpWindowSize` и аналогичных, игнорируемых со времен Vista.

## Использование

### 1. Применение профиля

1. Скачайте или скопируйте содержимое `universal.reg`.
2. Запустите файл от имени администратора.
3. Перезагрузите систему для инициализации изменений в драйвере `tcpip.sys`.

### 2. Верификация состояния

Для проверки текущих параметров стека выполните в CMD/PowerShell:

```bash
netsh int tcp show global

```

Для расширенного анализа используйте:

```powershell
netsh int tcp show supplemental
Get-NetTCPSetting -SettingName internet

```
