# 🚀 TCP/IP Optimizer Service

**Автоматическая оптимизация сетевого стека Windows 10/11 в реальном времени**

Инструмент для тонкой настройки параметров TCP/IP, который работает как фоновая служба и автоматически регулирует настройки сети на основе анализа запущенных процессов.

## ⚙️ Использование

### Быстрый старт

#### Шаг 1: Подготовка

1. **Скачайте архив** с проектом
2. **Распакуйте** в папку **без кириллицы и спецсимволов** в названии
   - ✅ Правильно: `C:\Tools\tcp-optimizer\`
   - ❌ Неправильно: `C:\Пользователь\Мои программы\`
3. Убедитесь, что у вас установлен **Python 3.8+**

#### Шаг 2: Установка зависимостей

Откройте командную строку от имени администратора и выполните:

```cmd
cd C:\путь\к\папке\tcp-optimizer
pip install -r requirements.txt
```

#### Шаг 3: Запуск

**Вариант А — Ручное применение стратегий:**

1. Нажмите правой кнопкой мыши на `general.bat`
2. Выберите **"Запуск от имени администратора"**
3. Выберите нужную стратегию из меню

**Вариант Б — Автоматическая служба:**

1. Нажмите правой кнопкой мыши на `service.bat`
2. Выберите **"Запуск от имени администратора"**
3. Выберите пункт **[1] Установить службу**
4. Служба будет автоматически оптимизировать сеть в реальном времени

---

## 🎯 Стратегии оптимизации

### 1. 🎮 Gaming (Low Latency)

**Для кого:** Игроки в онлайн-игры (CS2, Valorant, Apex Legends, Fortnite и др.)

**Что делает:**
- `TcpAckFrequency = 1` — немедленная отправка подтверждений
- `TCPNoDelay = 1` — отключение алгоритма Нагла (уменьшает задержку)
- `TcpDelAckTicks = 0` — без задержки подтверждений
- `SackOpts = 1` — включение Selective ACK

**Результат:** Минимальная задержка (ping), быстрая реакция на действия

---

### 2. 📥 Torrent (High Throughput)

**Для кого:** Пользователи торрентов (qBittorrent, uTorrent, Deluge)

**Что делает:**
- `TcpAckFrequency = 2` — баланс между подтверждением и throughput
- `TcpWindowSize = 64240` — увеличенное окно приёма
- `Tcp1323Opts = 3` — window scaling + timestamps
- `TcpMaxDataRetransmissions = 10` — агрессивные повторные передачи

**Результат:** Максимальная пропускная способность, эффективная загрузка

---

### 3. 📺 Streaming (Balanced)

**Для кого:** Просмотр видео (YouTube, Netflix, VLC, браузеры)

**Что делает:**
- `TcpAckFrequency = 1` — стабильная передача данных
- `TcpDelAckTicks = 1` — небольшая задержка для пакетной обработки
- `TcpWindowSize = 64240` — оптимизированное окно
- Сбалансированные параметры для плавного воспроизведения

**Результат:** Плавное видео без буферизации

---

### 4. 📹 Video Conference (Real-time)

**Для кого:** Видеоконференции (Zoom, Teams, Discord, Skype)

**Что делает:**
- `TcpAckFrequency = 1` — минимальная задержка
- `TCPNoDelay = 1` — отключение Нагла
- `DontFragment = 1` — запрет фрагментации пакетов
- Приоритет реальному времени

**Результат:** Чёткое аудио и видео без задержек

---

### 5. 📦 Downloads (Bulk Transfer)

**Для кого:** Загрузка игр и программ (Steam, Epic Games, Battle.net)

**Что делает:**
- `TcpAckFrequency = 2` — оптимизация для больших объёмов
- `TcpWindowSize = 64240` — увеличенное окно
- `TcpMaxDataRetransmissions = 10` — надёжная доставка

**Результат:** Быстрая загрузка больших файлов

---

### 6. 🔄 Windows Default (Сброс)

**Для чего:** Возврат к стандартным настройкам Windows

**Что делает:**
- Восстанавливает все параметры реестра к значениям по умолчанию
- Рекомендуется использовать перед удалением программы

## ❓ Частые вопросы (FAQ)

### ❓ Требуется ли подключение к интернету для работы?

**Нет.** Инструмент работает полностью офлайн, все настройки применяются локально через реестр Windows.

---

### ❓ Нужны ли права администратора?

**Да.** Изменение параметров TCP/IP требует доступа к системному реестру. Все скрипты должны запускаться от имени администратора.

---

### ❓ Как проверить, что настройки применены?

1. Запустите `service.bat` → пункт [6] **Диагностика**
2. Или выполните в командной строке:
   ```cmd
   netsh int tcp show global
   ```

---

### ❓ Служба не устанавливается / ошибка pywin32

**Решение:**
```cmd
pip uninstall pywin32
pip install pywin32
python -m pywin32_postinstall -install
```
Затем попробуйте установить службу снова через `service.bat`.

---

### ❓ Как удалить службу и вернуть всё как было?

1. Запустите `service.bat` от имени администратора
2. Выберите пункт [2] **Удалить службу**
3. Затем выберите пункт [6] **Диагностика** → или запустите `general.bat` → [6] **Windows Default**
4. Перезагрузите компьютер

---

### ❓ Можно ли добавить свои профили?

**Да.** Откройте `profiles/profiles.json` и добавьте новый профиль по аналогии с существующими:

```json
"my_custom_profile": {
  "name": "My Custom Profile",
  "description": "Описание",
  "processes": ["myapp.exe"],
  "settings": {
    "TcpAckFrequency": 1,
    "TCPNoDelay": 1
  },
  "priority": 10
}
```

---

### ❓ Как часто служба проверяет процессы?

По умолчанию — каждые **5 секунд**. Можно изменить в `profiles/profiles.json`:

```json
"service_config": {
  "check_interval": 3
}
```

---

### ❓ Влияет ли это на стабильность системы?

Настройки протестированы и используют допустимые значения реестра Windows. Однако:
- Всегда делайте точку восстановления перед изменением системных настроек
- Используйте функцию **Windows Default** для сброса
- Служба автоматически определяет активные процессы и применяет только релевантные настройки

---

### ❓ Поддерживается ли Windows 7/8?

**Нет.** Инструмент разработан для **Windows 10 и 11** (64-бит). Структура реестра и параметры TCP/IP в более старых версиях Windows отличаются.

---

### ❓ Где находятся логи?

Логи сохраняются в папке `logs/tcp_optimizer.log`. Просмотреть их можно через `service.bat` → пункт [7].

## 🔧 Технические детали

### Изменяемые параметры реестра

Параметры применяются по пути:
```
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\
```

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| `TcpAckFrequency` | Частота отправки ACK | 1-2 |
| `TCPNoDelay` | Отключение алгоритма Нагла | 0-1 |
| `TcpDelAckTicks` | Задержка отложенных ACK | 0-2 |
| `TcpMaxDupAcks` | Максимум дублирующих ACK | 2-6 |
| `SackOpts` | Selective ACK | 0-1 |
| `TcpWindowSize` | Размер окна приёма | 65535-64240 |
| `Tcp1323Opts` | RFC 1323 опции | 0-3 |
| `DefaultTTL` | Time To Live | 64-128 |
| `DontFragment` | Запрет фрагментации | 0-1 |
