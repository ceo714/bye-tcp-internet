# bye-tcp-internet

**Industrial-grade TCP/IP baseline profile for modern Windows 10/11 networking stacks.**

Проект представляет собой верифицированный профиль настроек реестра для сетевого стека `tcpip.sys`. Цель — оптимизация параметров Windows для обеспечения стабильной работы в современных сетях (2026) без использования устаревших, деструктивных или «плацебо» методов.

---

## Философия: Minimal • Measurable • Reversible

1. **Minimal (Минимализм):** Вносятся только те ключи, которые считываются драйвером `tcpip.sys` в актуальных сборках. Никакого «мусора» из эпохи Windows XP/7.
2. **Measurable (Измеримость):** Каждое изменение проверяется через системные инструменты (`netsh`, `Get-NetTCPSetting`). Эффект подтвержден технически, а не субъективно.
3. **Reversible (Обратимость):** Проект не модифицирует файлы. Применение `rollback.reg` полностью удаляет оверрайды, возвращая стек к заводским значениям (Hardcoded Defaults).

---

## Profiles

Проект предлагает два уровня оптимизации. Выберите один в зависимости от задач.

### 1. universal.reg (Baseline)

Универсальный профиль для рабочих станций и серверов. Удаляет искусственные лимиты системы.

* **Network Throttling:** Отключение сетевого ограничения при работе мультимедиа-сервисов.
* **Reliability:** Гарантированная активация PMTU Discovery, SACK и Window Scaling.
* **Socket Management:** Расширение диапазона Ephemeral портов и ускорение очистки `TIME_WAIT`.
* **DNS Stability:** Оптимизация кэширования негативных ответов и отключение ECN для лучшей совместимости.

### 2. gmvelocity.reg (Low-Latency)

Специализированный профиль для игровых сценариев. Включает в себя все настройки `universal.reg`, дополняя их:

* **MMCSS Priority:** Повышение приоритета сетевых задач в планировщике (профиль `Games`).
* **Advanced Throttling:** Полное устранение системных задержек при обработке пакетов.
* **Port Reuse Speed:** Оптимизация механизмов быстрого освобождения и повторного использования портов.
* **DNS Latency:** Снижение задержек резолвера при повторных попытках подключения.

## Technical Scope (Что НЕ изменяется)

Чтобы сохранить стабильность и адаптивность современного стека, проект **принципиально игнорирует** следующие категории:

* **Autotuning:** Не отключается (динамическое управление окном в Windows 11 эффективно по умолчанию).
* **Congestion Control:** CUBIC остается стандартным провайдером.
* **TCPNoDelay:** Не навязывается глобально (регулируется на уровне приложений).
* **Legacy Keys:** Не содержит `TcpWindowSize` и другие параметры, не поддерживаемые со времен Vista.

## Methodology & Safety

Проект базируется на анализе поведения `tcpip.sys` (2025–2026 гг.):

* **Прозрачность:** Изменения устраняют только клиентские ограничения, не пытаясь обойти физические законы задержки (RTT).
* **Совместимость:** Windows 10 (22H2+), Windows 11, Windows Server 2022/2025.
* **Безопасность:** Использование только `Registry Overrides` — самого безопасного способа тюнинга системы.

## Использование и верификация

### 1. Установка

1. Запустите выбранный `.reg` файл от имени администратора.
2. **Обязательно перезагрузите систему.**

### 2. Верификация

Для подтверждения изменений выполните в консоли:

```bash
# Проверка глобальных флагов
netsh int tcp show global

# Проверка параметров шаблона
powershell "Get-NetTCPSetting -SettingName internet"

```

### 3. Откат (Rollback)

Для возврата к исходному состоянию используйте `rollback.reg` и перезагрузите ПК. Это удалит все пользовательские оверрайды.
